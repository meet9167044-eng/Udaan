# UdaanScore — What to Do Now

## Goal
Finish the remaining work quickly and make the project demo-ready.

## Priority 1 — Make the app easier to understand
- Make sure every page has one clear purpose.
- Add one strong headline and one clear action button on each page.
- Remove confusing or extra UI elements that do not help the user.

## Priority 2 — Make the score feel believable
- Show clearly how the score is generated.
- Add a simple explanation for each score factor.
- Make the simulator show before/after changes clearly.
- Explain why a score goes up or down in plain language.

## Priority 3 — Improve the main flow
The main user flow should be:
1. Landing page
2. Login / role selection
3. Dashboard
4. Consent page
5. Simulator
6. Journey page

Each page should guide the user to the next step.

## What to do page by page

### Landing page
- Keep it short and simple.
- Show what UdaanScore does in 1–2 lines.
- Add clear buttons for Borrower, Lender, and Admin.
- Make the product value obvious immediately.

### Login page
- Make role selection feel clear and easy.
- Show what each role will experience after login.
- Keep the page simple and focused.

### Dashboard
- Show the trust score prominently.
- Add a short summary: current score, risk level, loan eligibility, next best action.
- Make the score factors easy to read.
- Add a simple “How to improve” section.

### Consent page
- Explain why consent matters.
- Show how consent affects the score and loan access.
- Add a short note on what happens if consent is revoked.

### Simulator
- Make it more practical and scenario-based.
- Show example actions such as paying bills on time, improving savings, or maintaining steady cash flow.
- Show the score change clearly after each action.
- Keep the language simple.

### Journey page
- Make it feel like a real action plan.
- Show the next best step clearly.
- Make milestone rewards and score impact easy to understand.

### Lender and Admin pages
- Keep them clean and decision-focused.
- Highlight borrower risk, fraud alerts, and important actions.
- Avoid unnecessary visual clutter.

## What to improve in the score logic
- Make the scoring explanation more transparent.
- Use simple labels for each input factor.
- Show score contribution per factor.
- Avoid making the scoring feel like magic.

## UI / UX checklist
- Use consistent spacing.
- Keep buttons and headings clear.
- Reduce unnecessary cards and text.
- Make important numbers stand out.
- Ensure the app looks polished on desktop and mobile.

## Demo checklist
Before presenting the project:
- Open the app and test all major pages.
- Make sure navigation works.
- Check that the score and simulator values appear correctly.
- Make sure the story is understandable in under 30 seconds.

## Final focus
The project should now be judged on:
- clarity
- usability
- believable score logic
- strong demo flow

## Immediate next step
Start with the dashboard, simulator, and consent page. These three pages matter most for the product story.
